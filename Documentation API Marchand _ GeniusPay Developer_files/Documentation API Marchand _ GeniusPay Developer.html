Authentification

Toutes les requêtes API doivent inclure vos clés dans les headers HTTP. Vous trouverez vos clés dans Paramètres → API.

Header Description
X-API-Key Votre clé publique (pk_sandbox_... ou pk_live_...)
X-API-Secret Votre clé secrète (sk_sandbox_... ou sk_live_...)
Content-Type application/json

Exemple avec cURL

curl -X POST https://pay.genius.ci/api/v1/merchant/payments \
  -H "X-API-Key: pk_sandbox_xxxxxxxx" \
  -H "X-API-Secret: sk_sandbox_xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"amount": 5000, "customer": {"phone": "+221771234567"}}'

Ne partagez jamais votre clé secrète

La clé X-API-Secret ne doit jamais être exposée côté client (frontend, app mobile). Gardez-la uniquement côté serveur.

Démarrage rapide

Créez un paiement et redirigez votre client vers notre page de checkout sécurisée. Deux lignes de code suffisent. 

// 1. Créer le paiement (sans spécifier payment_method)
$response = Http::withHeaders([
    'X-API-Key' => 'pk_live_xxx',
    'X-API-Secret' => 'sk_live_xxx',
])->post('https://pay.genius.ci/api/v1/merchant/payments', [
    'amount' => 15000,
    'description' => 'Commande #123',
]);

// 2. Rediriger vers la page de checkout GeniusPay
return redirect($response['data']['checkout_url']);

Rapide

Intégration en 2 lignes, aucune logique de paiement à gérer

Sécurisé

Page de checkout hébergée et sécurisée par GeniusPay

Tous les moyens

Wave, Orange, MTN, Moov, cartes bancaires

Comment ça marche ?

En omettant le paramètre payment_method, GeniusPay génère une URL vers la page de checkout. Le client y choisit son moyen de paiement préféré, effectue le paiement, puis est redirigé. Vous recevez un webhook de confirmation.

Comment utiliser les metadata

Le champ metadata vous permet de stocker vos propres données dans la transaction. Envoyez-les lors de la création (POST), puis récupérez-les avec ce endpoint (GET).

1. Envoyer les metadata (POST)

{
  "amount": 15000,
  "customer": {
    "name": "Amadou Diallo",
    "phone": "+221771234567"
  },
  "metadata": {
    "order_id": "CMD-2025-0042",
    "product_name": "Abonnement Premium",
    "user_id": "usr_abc123",
    "plan": "monthly"
  }
}

2. Récupérer les metadata (GET)

{
  "success": true,
  "data": {
    "reference": "MTX-A1B2C3D4E5",
    "status": "completed",
    "metadata": {
      "order_id": "CMD-2025-0042",
      "product_name": "Abonnement Premium",
      "user_id": "usr_abc123",
      "plan": "monthly"
    }
  }
}

Exemple d'intégration — PHP / Laravel

// 1. Créer le paiement avec vos metadata
$response = Http::withHeaders([
    'X-API-Key'    => 'pk_live_...',
    'X-API-Secret' => 'sk_live_...',
])->post('https://pay.genius.ci/api/v1/merchant/payments', [
    'amount'   => 15000,
    'customer' => ['name' => 'Amadou', 'phone' => '+221771234567'],
    'metadata' => [
        'order_id' => $order->id,
        'user_id'  => $user->id,
        'plan'     => 'premium',
    ],
]);

$reference = $response->json('data.reference');

// 2. Récupérer la transaction et vos metadata
$payment = Http::withHeaders([
    'X-API-Key'    => 'pk_live_...',
    'X-API-Secret' => 'sk_live_...',
])->get("https://pay.genius.ci/api/v1/merchant/payments/{$reference}");

if ($payment->json('data.status') === 'completed') {
    Order::find($payment->json('data.metadata.order_id'))->markAsPaid();
}

Exemple d'intégration — JavaScript / Node.js

// 1. Créer le paiement
const createRes = await fetch('https://pay.genius.ci/api/v1/merchant/payments', {
  method: 'POST',
  headers: {
    'X-API-Key': 'pk_live_...', 'X-API-Secret': 'sk_live_...',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    amount: 15000,
    customer: { name: 'Amadou', phone: '+221771234567' },
    metadata: { order_id: 'CMD-2025-0042', user_id: 'usr_abc123' },
  }),
});
const { data } = await createRes.json();

// 2. Récupérer par référence
const getRes = await fetch(
  `https://pay.genius.ci/api/v1/merchant/payments/${data.reference}`,
  { headers: { 'X-API-Key': 'pk_live_...', 'X-API-Secret': 'sk_live_...' } }
);
const payment = await getRes.json();
console.log(payment.data.metadata.order_id); // "CMD-2025-0042"

Bonnes pratiques metadata

  • Stockez votre order_id pour lier le paiement
  • Ajoutez un user_id pour identifier le client
  • Utilisez des clés descriptives (snake_case)
  • Les valeurs doivent rester simples (string, number)

Erreurs courantes

  • Ne pas envoyer les metadata au POST = rien à récupérer
  • Ne pas stocker la reference côté serveur
  • Confondre id (interne) et reference (MTX-...)
  • Oublier les headers d'authentification dans le GET

PawaPay — Mobile Money Afrique

PawaPay est un agrégateur de mobile money. GeniusPay intègre actuellement 12 pays africains avec 24 opérateurs MMO disponibles. Le système détecte automatiquement le pays depuis le numéro de téléphone.

Auto-routing

Envoyez le numéro, GeniusPay détecte pays et opérateur

Multi-devises

9 devises avec conversion automatique vers XOF

Choix explicite

Spécifiez mmo_provider pour cibler un opérateur

12 Pays

Bénin, Cameroun, CI, Congo, RDC, Gabon, Kenya, Rwanda, Sénégal, etc.

Guide d'intégration — 3 modes

Le numéro de téléphone suffit. GeniusPay détecte automatiquement le pays, l'opérateur et la devise. C'est le mode recommandé. 

curl -X POST https://pay.genius.ci/api/v1/merchant/payments \
  -H "X-API-Key: pk_live_xxx" \
  -H "X-API-Secret: sk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000,
    "payment_method": "pawapay",
    "customer": {
      "phone": "+221771234567"
    }
  }'

Auto-détection : Sénégal → Free/Orange Money → XOF. Aucune configuration supplémentaire requise.

Spécifiez directement le code opérateur pour forcer le routage vers un opérateur précis. 

{
  "amount": 5000,
  "currency": "XOF",
  "payment_method": "pawapay",
  "mmo_provider": "ORANGE_SEN",
  "customer": {
    "phone": "+221771234567",
    "country": "SN"
  }
}

Pour la RD Congo qui supporte 2 devises (CDF et USD), spécifiez la devise souhaitée. La conversion vers XOF est automatique. 

{
  "amount": 10,
  "currency": "USD",
  "payment_method": "pawapay",
  "customer": {
    "phone": "+243XX XXX XXXX",
    "country": "CD"
  }
}

Conversion automatique : 10 USD × 600 = 6 000 XOF stockés dans votre solde marchand.

Bonnes pratiques

  • Utilisez toujours le préfixe international (+221, +254, etc.)
  • Laissez GeniusPay détecter l'opérateur (meilleur taux de succès)
  • Pour les devises étrangères, spécifiez currency
  • Consultez GET /pawapay/providers pour lister dynamiquement les opérateurs
  • Les conversions sont transparentes : envoyez en devise locale, recevez en XOF

Metadata dans les webhooks

Vos metadata envoyées lors de la création du paiement sont incluses dans le payload webhook. Utilisez-les pour identifier la commande dans votre système.

Sécurité des webhooks

Vérifiez toujours la signature des webhooks pour vous assurer qu'ils proviennent bien de GeniusPay.

Important

Ne traitez jamais un webhook sans vérifier sa signature. Cela vous protège contre les requêtes malveillantes.

Headers webhook

Header Description
Content-Type application/json
X-Webhook-Signature Signature HMAC-SHA256
X-Webhook-Timestamp Timestamp Unix de l'envoi
X-Webhook-Event Type d'événement (ex: payment.success)
X-Webhook-Delivery ID unique de livraison (optionnel)
X-Webhook-Environment sandbox ou live
User-Agent GeniusPay-Webhook/1.0

Vérification de la signature

Format : signature = HMAC-SHA256(timestamp + "." + json_payload, secret)

// Récupérer les headers
$signature = $request->header('X-Webhook-Signature');
$timestamp = $request->header('X-Webhook-Timestamp');
$event = $request->header('X-Webhook-Event');

// Récupérer le payload JSON
$payload = $request->all();

// Construire la donnée à vérifier
$data = $timestamp . '.' . json_encode($payload);

// Calculer la signature attendue
$secret = 'whsec_xxxxxxxx'; // Votre clé secrète webhook
$expectedSignature = hash_hmac('sha256', $data, $secret);

// Vérifier la signature
if (!hash_equals($expectedSignature, $signature)) {
    return response()->json([
        'status' => 401,
        'detail' => 'Invalid signature'
    ], 401);
}

// Vérifier le timestamp (protection replay attack — 5 min)
if (abs(time() - (int)$timestamp) > 300) {
    return response()->json([
        'status' => 400,
        'detail' => 'Timestamp too old'
    ], 400);
}

// Traiter l'événement
switch ($event) {
    case 'payment.success':
        // Marquer la commande comme payée
        break;
    case 'payment.failed':
        // Notifier le client
        break;
}

Statuts des paiements

Chaque transaction passe par un cycle de vie défini. Voici les statuts possibles.

pending

En attente de paiement. Le client n'a pas encore payé.

processing

En cours de traitement par le gateway de paiement.

completed

Paiement réussi. Le montant a été crédité.

failed

Paiement échoué (solde insuffisant, erreur réseau, etc.).

cancelled

Paiement annulé par le client ou le marchand.

refunded

Paiement remboursé au client.

Codes d'erreur

Toutes les erreurs API suivent un format cohérent avec un code machine et un message lisible.

Format d'erreur

{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Description lisible de l'erreur"
  }
}

Exemples d'intégration

Deux approches pour intégrer GeniusPay : le mode checkout (recommandé) et le mode direct.

Recommandé

Mode Checkout (2 lignes)

Sans payment_method, le client choisit son moyen de paiement sur la page GeniusPay. 

// Intégration en 2 lignes
$response = Http::withHeaders([
    'X-API-Key' => 'pk_live_xxx',
    'X-API-Secret' => 'sk_live_xxx',
])->post('https://pay.genius.ci/api/v1/merchant/payments', [
    'amount' => 15000,
    'description' => 'Commande #123',
]);

return redirect($response['data']['checkout_url']);
const response = await fetch('https://pay.genius.ci/api/v1/merchant/payments', {
  method: 'POST',
  headers: {
    'X-API-Key': 'pk_live_xxx',
    'X-API-Secret': 'sk_live_xxx',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    amount: 15000,
    description: 'Commande #123',
  }),
});

const { data } = await response.json();
window.location.href = data.checkout_url;
import requests

response = requests.post(
    'https://pay.genius.ci/api/v1/merchant/payments',
    headers={
        'X-API-Key': 'pk_live_xxx',
        'X-API-Secret': 'sk_live_xxx',
    },
    json={
        'amount': 15000,
        'description': 'Commande #123',
    }
)

checkout_url = response.json()['data']['checkout_url']
# Rediriger le client vers checkout_url
final payment = await geniusPay.checkout(
  amount: 15000,
  description: 'Commande #123',
);
launchUrl(Uri.parse(payment.checkoutUrl!));
Alternatif

Mode Direct (méthode spécifiée)

Avec payment_method spécifié, le client est redirigé directement vers le gateway. 

$response = Http::withHeaders([
    'X-API-Key' => 'pk_live_xxx',
    'X-API-Secret' => 'sk_live_xxx',
])->post('https://pay.genius.ci/api/v1/merchant/payments', [
    'amount' => 15000,
    'payment_method' => 'wave',
    'customer' => [
        'phone' => '+221771234567',
    ],
]);

return redirect($response['data']['payment_url']);
const response = await fetch('https://pay.genius.ci/api/v1/merchant/payments', {
  method: 'POST',
  headers: {
    'X-API-Key': 'pk_live_xxx',
    'X-API-Secret': 'sk_live_xxx',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    amount: 15000,
    payment_method: 'wave',
    customer: { phone: '+221771234567' },
  }),
});

const { data } = await response.json();
window.location.href = data.payment_url;
